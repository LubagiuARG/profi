# Backlog

Tareas vivas del proyecto. Punto único de verdad para qué hay que hacer.

## Cómo se organiza

| Archivo | Qué tiene |
|---------|-----------|
| [`tareas-backend.md`](./tareas-backend.md) | API, DB, scraper, MP, auth, infra |
| [`tareas-frontend.md`](./tareas-frontend.md) | UI, UX, copy, SEO, mobile |
| [`tareas-producto.md`](./tareas-producto.md) | Legales, marketing, AdSense, soporte |

## Cómo se priorizan

Cada tarea tiene una etiqueta:

- **🔴 P0** — bloqueante para el MVP, no se lanza sin esto
- **🟠 P1** — importante pero no bloquea el lanzamiento
- **🟡 P2** — mejora notable, no urgente
- **🟢 P3** — nice to have, fase 2+

## Cómo se trabaja

1. Tomar una tarea P0 antes de mirar P1+.
2. Marcar como `🟦 EN CURSO — @nombre` cuando arrancás (para que el otro no la tome).
3. Al terminar: tachar (`~~tarea~~`) o mover a una sección "Hechas" al final del archivo.
4. Si una tarea se descubre en el medio del trabajo, sumarla al archivo correspondiente con prioridad estimada.

## Origen del backlog inicial

La primera tanda de tareas vino del análisis del estado actual del código (abril 2026). Las inconsistencias detectadas en código están marcadas con 🐛.
